Playback contols are critically important for any embedded video player. Without them the user will become very frustrated with the video. Below, you will find a list of functions to allow playback contol of your embedded player.

Playing a Video

The primary functions for video playback are player.playVideo():Void, player.pauseVideo():Void, player.stopVideo():Void, and player.seekTo(seconds:Number, allowSeekAhead:Boolean):Void. These functions do exactly what you expect; play, pause, stop, and the video.

player.playVideo() will play the currently loaded/cued video.

player.pauseVideo() pauses the video that is currently playing uness the player is in the ended state.

player.stopVideo() will stop playback and cancel the current video from loading. Be careful when using this function. Using the stopVideo function will deactivate your video player. If you want to stop the current video from playing, use the pauseVideo function. If your intent is to load a new video, you can call one of the queuing functions without calling player.stopVideo().

player.seekTo(seconds:Number, allowSeekAhead:Boolean) queues the video to a specific time in the video. If the video is paused when this function is called iy will continue to be paused. If the video is in any other state it will play. seconds is the video time in seconds that the video will be advanced to. The allowSeekAhead parameter determines whether or not the player will allow the user to advance time past the cyrrent buffered time. If this is is true and the user picks a playtime past the point of buffering, the playe will make another request to the server.

Playing a Video in a Playlist

Videos that are part of a playlist use the functions above for their basic controls, but have another set of functions to move through the playlist.

player.nextVideo():Void loads and plays the next video in a playlist.

player.previousVideo():Void loads and plays the next video in a playlist.

player.playVideoAt(index:Number):Void loads and plays the video at a specified index in a playlist. Note that the indexes start at 0 and increment from there.
