In order to embed a YouTube player in your webpage, you will be using the IFrame API. This will allow you more flexibility than the flash or JavaScript APIs which required you to embed a flash object. The IFrame API instead uses the <iframe> tag to embed the video player, and Javascript to do things like queue videos for playback or adjust the volume.

The sections below will help guide you through using the IFrame API

Requirements

The HTML5 postMessage feature must be supported by the browser. This will alow scripts that are on your machine to run on the videos located on the YouTube server without using the same protocol, port number, etc.

The embedded player must be at least 200px by 200px, not including the control bar at the bottom. The YouTube recommended size for 16:9 players is 480px by 270px or larger.

Any web page that uses the IFrame API must implement the onYouTubeIframeAPIReady function. This function will be called after the page has finished downloaded the JavaScript for the player and will enable you to use the the API on your page.

Loading a Video Player

You can load a YouTube video player in two ways. The first is to use the <iframe> tag yourself, and the second is to use JavaScript.

When using the <iframe> tag, you must specify all of your parameters inside the tag. This includes the height and width of the player, the source location of the video, and the frameborder. Below is an example.

<iframe id="player" type="text/html" width="640" height="390" src="http://www.youtube.com/embed/VideoID?enablejsapi=1&origin=http://yourWebPage.com" frameborder="0"></iframe>
  • width refers to the width of the video player
  • height refers to the height of the video player
  • src contains the id of the video you wish to embed. It is also a good idea to include the origin parameter which specifies http vs https and the full domain of your host page.

If you would rather use JavaScript to create your YouTube player, then the function would like the following:

// This code loads the IFrame Player API code asynchronously.
var tag = document.createElement('script');
tag.src = "https://www.youtube.com/iframe_api";
var firstScriptTag = document.getElementsByTagName('script')[0];
firstScriptTag.parentNode.insertBefore(tag, firstScriptTag);

// This function creates an <iframe> (and YouTube player)
// after the API code downloads.
var player;
function onYouTubeIframeAPIReady() {
player = new YT.Player('player', {
height: '390',
width: '640',
videoId: 'ID of your video',
events: {
'onReady': onPlayerReady,
'onStateChange': onPlayerStateChange
}
});
}
Code above credit https://developers.google.com/youtube/iframe_api_reference
  • The first section of this code loads the JavaScript API code and uses DOM modification to do it asynchronously.
  • The second section of this code creates will create the YouTube player using the global variable "player".
